# Industry OS 🔥

**Автономная бизнес-операционная система на Hermes Agent.** Пять слоёв production-ready автоматизации — от DevOps-ядра до управления роем агентов и бизнес-аналитики.

Построено на [Hermes Agent](https://hermes-agent.nousresearch.com/). Проверено в бою: ценовой мониторинг, партнёрская аналитика, финансовая разведка, тендерный анализ.

## Стек

```
Слой 4: Разведывательные пайплайны  ← автономное исследование рынка, тендеры, финансовая криминалистика
Слой 3: Бизнес-ОС                  ← сортировка Drive, CRM, 1С-пайплайн, аналитика партнёров, мониторинг цен
Слой 2: Управление роем            ← реестр агентов, 4 линии, sweeper, human gate
Слой 1: DevOps-ядро                ← quality gates, ratchet loop, автообучение, граф зависимостей
Слой 0: Hermes Agent               ← skills, cron, delegate_task, memory, инструменты
```

## Что внутри

### DevOps Core (этот репозиторий)
| Скилл | Что делает |
|-------|-----------|
| **quality-gates** | 4-ступенчатая валидация (корректность → качество → безопасность → консистентность). Блокирует плохой вывод до продакшена. |
| **ratchet-loop** | Автономный цикл улучшения: улучшить → запустить → оценить → сохранить/откатить. |
| **auto-learning-pipeline** | 5-стадийный куратор: обнаружить → оценить → ранжировать → сохранить → выучить паттерны. |
| **skill-dependency-graph** | Визуализация зависимостей скиллов, обнаружение циклов, предложение композиций. |

### Swarm Management → [tera-harness](https://github.com/bytheby72/tera-harness)
| Компонент | Что делает |
|-----------|-----------|
| Run Registry | SQLite-реестр запусков агентов с hashline-цепочкой аудита |
| 4 линии | scout (мониторинг) → research (анализ) → fulfill (исполнение) → delivery (доставка) |
| Recovery Sweeper | Авто-перезапуск зависших агентов, эскалация после 3 отказов |
| Human Gate | Единственная точка участия человека — агент предлагает, человек утверждает |

### Бизнес-ОС → [anatoli-google-infra](https://github.com/bytheby72/anatoli-google-infra)
| Подсистема | Что делает |
|-----------|-----------|
| Drive Sorter | Авто-классификация файлов в 49-папочную структуру |
| Obsidian CRM | Markdown-база людей/компаний/встреч |
| 1С-пайплайн | Email → парсинг → SQLite для фин. отчётов |
| Партнёрская аналитика | Сравнение объёмов, детекция рисков, рекомендации |
| Мониторинг цен | Camoufox-отслеживание цен конкурентов с самовосстановлением |

### Разведывательные пайплайны
| Пайплайн | Что делает |
|----------|-----------|
| Рыночные мониторы | Непрерывный скрапинг цен/каталогов конкурентов |
| Тендерная разведка | Парсинг гос. тендеров, скоринг возможностей |
| Финансовая криминалистика | Парсинг банковских выписок, детекция банкротств |

### Методология → [hermes-loop-skills](https://github.com/bytheby72/hermes-loop-skills)
Loop engineering с 4 hardening-скиллами: audit, harden, deploy, security.

## Принципы проектирования

1. **Namespace-first** — каждый скилл в чётком неймспейсе
2. **Database-driven** — SQLite для всего долгосрочного состояния
3. **Local-first** — конфиденциальные данные не покидают инфраструктуру пользователя
4. **Fail-closed** — неопределённое состояние останавливает систему
5. **Self-documenting** — каждый скилл с pitfalls, уроками, процедурами восстановления

## Эволюция

- **Фаза 1 (готово):** DevOps Core + Бизнес-ОС
- **Фаза 2 (в процессе):** Swarm Management через Tera Harness
- **Фаза 3 (далее):** Автономные разведывательные пайплайны с human gate
- **Фаза 4 (будущее):** Самоулучшающаяся ОС — ratchet loop применяется к самой ОС
